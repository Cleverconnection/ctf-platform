Notas internas do proxy

Este proxy foi criado para facilitar integrações com armazenamentos S3 externos. Ele encaminha requisições informadas pelos analistas e devolve o corpo sem armazenar cópias.

Checklist operacional

  • URLs externas devem apontar para objetos versionados.
  • O proxy mantém compatibilidade com endpoints internos como http://127.0.0.1:8080/s3/<bucket>.
  • O bucket sensível atual é {{ bucket }}; utilize apenas chaves autorizadas.

Riscos conhecidos

Não há validação rígida do host de destino. Caso alguém informe URLs internas por engano, o proxy tentará buscá-las a partir da rede corporativa. O time de segurança está revisando filtros adicionais.

O parâmetro forward_headers controla se o proxy envia cabeçalhos padrão. Use com cautela ao depurar integrações com buckets privados.
